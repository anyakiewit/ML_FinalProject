\paragraph{SemEval-2024 Task 8 and Subtask C}
Our research is within in the framework of SemEval-2024 Task 8, which benchmarks Multigenerator, Multidomain and Multilingual Machine-Generated Text Detection.
Subtask C presents the challenge of boundary detection in mixed texts. The objective is to identify the word index where a document transitions from human-written to machine-generated content.
Evaluating this requires models to identify abrupt shifts in stylistic consistency and token predictability.

\paragraph{Stylometry and Probabilistic AI Text Detection}
Historically authorship attribution and text classification relied heavily on stylometric features to establish an author's unique fingerprint \cite{neal2017surveying}.
Surface-level markers such as average word length, punctuation frequency and capitalization patterns have proven to be robust discriminators of human writing styles \cite{neal2017surveying}.

However with the rise of Large Language Models (LLMs), detection mechanisms have increasingly adopted probabilistic approaches.
Masked Language Models (MLMs) are now frequently used to extract predictive features such as token log-probability, rank and perplexity.
Recently, Xu and Sheng (2024) demonstrated this approach by utilizing MLM-based perplexity to detect AI-generated sequences \cite{xu2024detecting}.
Their findings emphasize that AI-generated text, in their case code, consistently exhibits lower and more uniform perplexity compared to the natural variance of human authorship \cite{xu2024detecting}.
% This validates our use of both stylometric baseline features and MLM-derived metrics to identify the exact boundary where text shifts from human variance to machine predictability.

\subsection{Dataset Description}

\textcolor{red}{Written texts where each token separated by space is either labeled with 0 for Human written or Machine written text}
\\ \\
\textcolor{red}{Statistics of the data}
\\ \\
The data for this study is provided by the SemEval-2024 Task 8 organizers for Subtask C (Human-Machine Mixed Text Detection).
The dataset consists of documents where the first part is written by a human and the following part is generated by a LLM.
The data is structured in JSON Lines (JSONL) format, where each entry contains a unique id, the raw text, and a list of labels.
The label represents whether the token at the 0-based index is human-written (0) or machine-written (1).

\section{Method}

\subsection{Pipeline}

\textcolor{red}{small summary of the task, what we are trying to predict}

For each instance, a context window is constructed around a target word.
Based on this context, a set of statistical features was extracted to capture surface-level charactetristics of the text.
These were then appended to the MLM features to create a comprehensive feature vector for each instance.
This feature vector is then used to train a classifier to predict whether the target word is human-written or machine-generated.

\subsection{Preprocessing}

\textcolor{red}{Split only on (" "), lemmatization/stemming and punctuation handling kept to a minimum}

The dataset was split into a training set and a test set, with an 80/20 split.
The data was tokenized by splitting on spaces, to ensure the labeling remained accurate.

\subsubsection{Statistical Features}

Based on a context window, a set of statistical features was extracted to capture surface-level characteristics of the text.

These features include: (1) the average word length within the context window, (2) the number of punctuation marks, (3) a binary indicator capturing whether the target word is written in all capital letters, (4) a binary indicator whether the target word has a capital letter as its first letter, and (5) the tf-idf score of the target word.
To ensure that these features reflect only meaningful textual information, padding tokens (\texttt{<PAD>}) were excluded from all computations.

\subsubsection{MLM Features}

% \textcolor{red}{Describe what model used and why. What the effect of this is and how the results are to be interpreted}

To capture the difference between human-written and machine-generated text we used a Masked Language Model (MLM) to extract features
that capture the predictability of tokens.
For this we chose to use the pre-trained \texttt{roberta-large} model \cite{liu2019roberta}. For each context window, we mask the
central target token using the tokenizer's \texttt{mask} token and pass the sequence through the model.
We then extract the probability of the actual target token and its rank within the predicted probability distribution.

Our choice for \texttt{roberta-large} is motivated by its bidirectional architecture, which processes both left and right context at the
same time.
Recent studies have shown that bidirectional models are better at discriminating between human-written and machine-generated text than
unidirectional (autoregressive) models \cite{huang-etal-2024-ai}.
\texttt{roberta-large} has also been used in top-performing systems in the SemEval-2024 Task 8 due to its robust ability to capture deep
semantic cues and structural predictability \cite{datta2024weighted}.

This approach is based on the hypothesis that language models generate highly predictable text.
Machine-generated tokens should therefore show higher probabilities and better (lower) ranks than human-written text,
which contains more variance.

An analysis of the extracted features (Table \ref{tab:mlm_stats}) supports our hypothesis.
Across the training set, machine tokens were very predictable, with a median rank ($\tilde{r}$) of 1.0 and a
median probability ($\tilde{p}$) of 0.310. In contrast, human tokens displayed lower predictability ($\tilde{p}=0.151$).
The higher mean rank for human text ($\bar{r}=295.4$) compared to machine text ($\bar{r}=218.6$)
illustrates the "long tail" of natural human writing, showing the creative and idiosyncratic word choices that language
models rarely predict.

\subsubsection{SVM}

\textcolor{red}{Describe the experiments}

SVMs are maximal margin classifiers, they optimize the decision boundary by finding the hyperplane that maximizes the distance between
the nearest data points (support vectors) of different classes.

As standard SVMs can be slow when dealing with very large datasets and we are analyzing hundreds of thousands of individual words,
we experimented with two faster, highly scalable versions of the model:
a Linear SVM (\texttt{LinearSVC}) and an SVM optimized for large-scale data (\texttt{SGDClassifier}).

For the Linear SVM we ran an automated search to fine-tune the model's parameters, to find its most accurate settings.

To measure the impact of our different features, we trained and evaluated these models twice:
first using only the basic text statistics, and then adding the MLM predictability scores
to see how much the context helped the model's accuracy.

\paragraph{SVM Performance and Feature Importance}
Adding MLM features improved both \texttt{LinearSVC} and \texttt{SGDClassifier}.
Using basic text statistics alone, the best model (SGD) achieved 0.6947 accuracy and 0.6646 Macro F1.
Incorporating MLM features boosted this to 0.7110 accuracy, 0.6786 Macro F1, and 0.83 precision for machine-generated tokens.

Figure \ref{fig:svm_feat_importance} explains this improvement: while basic models rely on \texttt{tfidf\_score} and \texttt{average\_word\_length}, the combined model shifts almost entirely to MLM features, proving them to be much stronger indicators of AI-generated text.
